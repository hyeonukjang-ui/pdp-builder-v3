# PDP 전환 카피 가이드라인

> 사용자의 구매 결정을 설계하는 카피 원칙
> Google "Messy Middle" + 행동 심리학 기반

---

## 핵심 전환: "설명하기"가 아닌 "결정하게 하기"

사용자는 PDP에서 **긴 설명을 읽으러 온 게 아니다**. 5가지 질문에 답을 찾으러 왔다:

| 사용자 질문 | 답해야 하는 곳 | 카피 목적 |
|-----------|-------------|---------|
| "이거 괜찮은 거야?" | 리뷰 별점, UGC | 사회적 증거 |
| "핵심이 뭐야?" | 하이라이트 3~5개 | 빠른 판단 |
| "정확히 뭐가 포함이야?" | 포함/불포함 | 불안 제거 |
| "취소하면 어떻게 돼?" | 신뢰 배지, CTA 옆 | 리스크 해소 |
| "지금 사도 돼?" | 가격, 잔여석, 즉시확정 | 행동 유도 |

---

## Google "Messy Middle" 6가지 인지 편향 활용

| 편향 | 카피에서의 적용 | 예시 |
|------|--------------|------|
| **Category Heuristics** | 핵심 스펙을 숫자로 즉시 전달 | "약 5시간 · 한국어 · 최대 8명" |
| **Social Proof** | 다른 사람의 행동을 보여줌 | "4,200명이 선택한 투어" |
| **Authority** | 전문성/경력을 명시 | "현지 12년 거주 전문 가이드" |
| **Power of Now** | 즉시성 강조 | "즉시 확정 · 오늘 예약 가능" |
| **Scarcity** | 제한된 자원 표시 | "이 날짜 남은 자리 3석" (실시간 데이터만) |
| **Power of Free** | 무료 혜택 강조 | "48시간 전 무료 취소" |

## 블록별 카피 원칙

### Hero — 3초 안에 잡기

**원칙**: 제목은 "이게 뭔지" + "왜 사야 하는지"를 동시에.

```
패턴 A: {숫자}+{경험}
  "이스탄불에서 꼭 해야 할 3가지, 하루 만에"

패턴 B: {사회적 증거}+{경험}
  "4,200명이 선택한 바르셀로나 워킹투어"

패턴 C: {핵심 혜택}+{경험}
  "줄 서지 않고 입장하는 사그라다 파밀리아"
```

**금지**: "최고의", "완벽한" 같은 검증 불가 최상급. 대신 숫자와 구체적 혜택.

---

### trustBadges — 불안 해소 한 줄

```
✓ 48시간 전 무료 취소    (리스크 해소)
✓ 즉시 확정              (확실성)
✓ 최저가 보장            (가격 확신)
✓ e-티켓 · 프린트 불필요  (편의)
```

**원칙**: 동사 불필요. 명사구로 짧게. 아이콘 + 한 줄.

---

### Highlights — 구매 동기 3~5개

**원칙**: **구체적 대상 + 행동 동사 포함**, 숫자/장소명 필수, 최대 40자.

> "동사로 시작"이 아니라 "행동 동사를 포함"이 핵심. 구체적 대상(장소, 혜택)으로 시작하고 동사로 마무리해도 좋다.

```
좋은 예:
✓ 블루모스크 2만 장 이즈닉 타일을 전문 가이드와 감상
✓ 호텔 픽업/샌딩 포함, 이동 걱정 제로
✓ 인생샷 포인트에서 가이드가 직접 촬영

나쁜 예:
✗ 아름다운 이스탄불을 만끽하세요 (모호, 구체적 대상 없음)
✗ 최고의 가이드와 함께하는 투어 (검증 불가 최상급)
✗ 다양한 장소를 방문합니다 (구체성 없음, 번역체)
```

---

### Overview — 접혀있는 상태에서도 의미 있게

**원칙**: 첫 3줄이 가장 중요 (기본 접힘 상태에서 보이는 부분).
- 1줄: 이 상품의 핵심 한 문장
- 2~3줄: 차별점 또는 핵심 경험
- 4줄~: "더 보기"로 펼쳤을 때만 보이는 상세

```
[보이는 부분]
이스탄불의 심장부, 올드 시티를 하루 만에 깊이 있게 만나는 프라이빗 투어입니다.
유네스코 세계문화유산인 블루모스크와 아야소피아를 전문 가이드의 해설과 함께 둘러보며,
비잔틴과 오스만 두 제국의 흔적을 동시에 느낄 수 있습니다.

[더 보기 후]
관광객으로 붐비는 루트 대신, 현지인만 아는 골목길과 로컬 카페를 경유하는...
```

---

### Inclusions — 혜택 프레이밍

**원칙**: 포함 항목은 "혜택"으로, 불포함 항목은 "대안"과 함께.

```
✅ 포함
  한국어 전문 가이드 — 12년 경력, 터키 역사학 전공
  전용 차량 이동 — 호텔 픽업/샌딩 포함
  블루모스크 · 아야소피아 입장료

❌ 불포함
  점심 식사 — 💡 가이드가 현지 맛집 안내 (1인 약 15,000원)
  가이드 팁 — 💡 선택사항 (관례상 1인 $5~10)
```

---

### CTA — 저압박 + 신뢰

**원칙**: "사라"가 아니라 "확인하라". 행동의 심리적 비용을 낮춘다.

| 카테고리 | CTA 문구 | 근거 |
|---------|---------|------|
| 투어 | "일정 확인하기" | 날짜 선택이 첫 관문 |
| 티켓 (테마파크) | "옵션 선택하기" | 옵션별 가격 비교가 핵심 |
| 티켓 (교통) | "가격 보기" | 단순 구매, 가격 확인이 핵심 |
| 티켓 (시티패스) | "절약 금액 보기" | 가격 앵커링, "이득인가?" 해소 |
| 티켓 (체험) | "날짜 확인하기" | 공연/유람선은 날짜·시간이 핵심 |
| 서비스 | "예약 가능일 보기" | 스냅/스파/미식은 날짜 선택 |
| 액티비티 | "예약 가능일 보기" | 가용성 확인이 핵심 |
| 세미패키지 | "견적 확인하기" | 복잡한 가격 구조 |

**CTA 옆 텍스트**: "48시간 전 무료 취소" (Power of Free 활용)

---

### FAQ — 실제 데이터 기반 자동 생성

**원칙**: 상위 CS 문의 데이터 + 상품 데이터에서 자동 매핑.

| 트리거 데이터 | 질문 | 답변 패턴 |
|-------------|------|----------|
| cancel_policy 존재 | "취소/환불은 어떻게 되나요?" | "{기한} 전 무료 취소 가능해요." |
| meeting_point 존재 | "어디서 만나나요?" | "{장소명}에서 만나요. {교통편}" |
| 연령 제한 있음 | "아이도 참여할 수 있나요?" | "{연령} 이상 참여 가능해요." |
| language 존재 | "한국어 가능한가요?" | "{언어} 가이드가 동행해요." |
| duration 존재 | "얼마나 걸리나요?" | "총 약 {시간} 소요돼요." |

---

### Social Proof — 반드시 실제 데이터만

**원칙**: 가짜 긴급성은 신뢰를 파괴한다. **실시간 API 데이터만 사용**.

```
허용:
"오늘 23명이 이 상품을 예약했어요" (실제 예약 건수)
"이 날짜 남은 자리 3석" (실제 재고)
"지난 1시간 동안 5명이 보고 있어요" (실제 조회 데이터)

금지:
"마감 임박!" (기준 불명확)
"한정 수량!" (실제 한정이 아닌 경우)
"지금 안 사면 후회!" (과장)
```

---

### Reviews — 리뷰 카피

리뷰 자체는 사용자 생성 콘텐츠지만, **시스템이 생성하는 카피**가 있다:

```
리뷰 있을 때:
  AI 요약: "여행자들이 가장 많이 언급한 키워드: 친절한 가이드, 알찬 코스, 사진 포인트"
  → 실제 리뷰 데이터에서 추출. 주관적 판단 금지.

리뷰 없을 때:
  "아직 첫 번째 리뷰를 기다리고 있어요"
  → 부정적으로 느끼지 않도록 중립~긍정 톤. "리뷰가 없습니다" (X)
  → 대안: 예약 건수, 판매자 평점 등 다른 사회적 증거로 보완
```

## 상품 유형별 톤앤매너

| 구분 | 투어 | 티켓 | 서비스 | 액티비티 | 세미패키지 |
|------|------|------|--------|----------|----------|
| 핵심 감정 | 안심, 편안함 | 실용, 합리 | 감성, 특별함 | 설렘, 도전 | 편의, 가성비 |
| USP 강조 | 가이드 전문성, 이동 편의 | 빠른 입장, 가격 | 결과물 품질, 전문가 포트폴리오 | 안전, 초보 OK | 항공+숙박 올인원 |
| 감각 언어 비중 | 중간 (장소 묘사) | 낮음 (스펙/조건) | 높음 (분위기, 감성 묘사) | 높음 (체험 묘사) | 중간 |
| CTA | "일정 확인하기" | "가격 보기" | "예약 가능일 보기" | "예약 가능일 보기" | "견적 확인하기" |

## 모바일 카피 원칙 (v2.0 신규)

모바일 사용자는 화면이 작고 집중 시간이 짧다. 데스크톱 카피를 그대로 쓰면 안 된다.

| 원칙 | 가이드 | 예시 |
|------|--------|------|
| **글자 수 제한** | 제목 25자 이내, 하이라이트 40자 이내 (모바일 1줄 기준) | "이스탄불 올드시티 프라이빗 투어" (17자) |
| **첫 단어가 핵심** | 스캔 시 첫 2~3단어만 읽힘. 핵심 정보를 앞에 배치 | "무료취소 가능 · 48시간 전까지" (O) / "48시간 전까지 무료취소가 가능합니다" (X) |
| **줄바꿈 활용** | 긴 문장 대신 짧은 줄 여러 개. 스캔 용이 | 포함사항을 문장이 아닌 리스트로 |
| **숫자 우선** | 텍스트보다 숫자가 모바일에서 시각적 우위 | "약 5시간" > "약 다섯 시간 정도 소요" |
| **터치 친화 텍스트** | 링크/버튼 텍스트는 터치하기 쉬운 길이 (3~5단어) | "일정 확인하기" (O) / "지금 바로 일정을 확인해보세요" (X) |

## 카피 품질 체크리스트

자동 생성된 카피 검수 시:

- [ ] **검증 불가 최상급 없는가** ("최고의", "완벽한", "압도적인")
- [ ] **구체적 숫자가 있는가** (시간, 인원, 거리, 가격)
- [ ] **하이라이트에 구체적 대상 + 행동 동사가 포함되어 있는가**
- [ ] **불포함 항목에 대안이 있는가** (단순 나열 X)
- [ ] **CTA가 저압박인가** ("예약하기" → "날짜 확인하기")
- [ ] **사실과 다른 정보 없는가** (AI 환각 체크)
- [ ] **가짜 긴급성 없는가** (실시간 데이터 기반만 허용)
- [ ] **모바일 1줄에 잘리지 않는가** (제목 25자, 하이라이트 40자 이내)
