# Phase 1: 진단 — PDP Health Score + 우선순위

> 오너: 박윤경 | 기간: 2주 + 버퍼 1주 (4월 1~3주차) | 상태: 대기

---

## 목표

659개 상품 중 "어디에 먼저 콘텐츠를 생성할지" 데이터 기반 결정. 콘텐츠 부재 상품을 진단하고 우선순위를 매긴다.

---

## 핵심 질문

> 257개 콘텐츠 임계점 미달 상품 중, AI 콘텐츠 생성의 ROI가 가장 높은 30개는?

```
우선순위 = 낮은 Health Score × 높은 매출 (1사분면)
→ "콘텐츠는 부실한데 트래픽은 있는 상품" = 생성 ROI 최대
```

---

## 태스크 체크리스트

- [ ] **1-1** BigQuery에서 상품별 블록 존재 여부 추출 (기존 분석 쿼리 확장)
- [ ] **1-2** Health Score V1 산출 로직 구현 (카테고리별 필수 블록 Y/N × 일매출)
- [ ] **1-3** 일매출 데이터와 조인 → 우선순위 매트릭스 완성
- [ ] **1-4** 상위 30개 상품 일평균 PDP뷰 + 현재 CVR 확인 (A/B 표본 크기 산정용)
- [ ] **1-5** 상위 30개 선정 + 부족 항목 리포트 (기원/현욱 작업 지시서 역할)

## Health Score V1 산출 로직

> 정본: [00-project-overview.md](../00-project-overview.md) — Health Score 개념 정의
> 구현: `lib/health-scorer.js` — 카테고리별 필수 블록 + Score/Grade 산출
> 상세: [02-pdp-section-system.md](../02-pdp-section-system.md) — 카테고리별 필수 블록 목록

```
Score = (보유 필수 블록 수 / 카테고리 필수 블록 총수) × 100
Grade: A(90+) / B(70~89) / C(50~69) / D(0~49)
```
