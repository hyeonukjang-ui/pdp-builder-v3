# Phase 2: AI 콘텐츠 생성 + 전환 레시피 적용 (30개 상품)

> 오너: 박윤경(설계) + 기원(AI 생성) + 현욱(빌더) | 기간: 3주 + 버퍼 1주 (4월 4주차 ~ 5월 3주차) | 상태: 대기

---

## 목표

상위 30개 상품에 AI로 이미지+카피를 생성하고, 카테고리별 전환 레시피에 맞춰 PDP를 완성한다. A/B 테스트 가능 상태까지.

---

## 핵심 프로세스

```
1. 상품 데이터 수집 (URL/API → raw data)
2. AI 카피 생성 → 상세: 11-ai-copy-strategy.md
3. AI 이미지 생성 (상품 특성에 맞는 비주얼) → TBD: 기원 담당
4. 전환 레시피 적용 → 상세: 02-pdp-section-system.md
5. 빌더에서 조립 + QA
6. 유저빌리티 테스트
```

---

- [ ] **2-5** 30개 상품 이미지 생성 (hero, imageGrid 등)

- [ ] **2-6** trustBadges + socialProof 블록 빌더 추가 (JSON 스키마 사전 공유)
- [ ] **2-7** 빌더에서 블록 배치 + QA

## AI 파이프라인

> **카피 가이드 정본**: [06-copy-guidelines.md](../06-copy-guidelines.md)

### 이미지 생성

| 블록 | 이미지 유형 | 생성 방식 |
|------|-----------|----------|
| hero | 대표 이미지 (16:9) | AI 생성 or 기존 이미지 최적화 |
| imageGrid | 갤러리 이미지 (4~8장) | AI 생성 + 기존 이미지 혼합 |
| itinerary | 스톱별 이미지 | 장소 기반 AI 이미지 생성 |
| guideProfile | 가이드 프로필 | 기존 데이터 or 기본 아바타 |

> **이미지 생성 파이프라인**: 기원 담당, Phase 2 시작 전까지 스펙 정의 필요
