## Phase 0: 방향 합의 + ROI 추정 (현재)

| 항목 | 내용 |
|------|------|
| **목표** | TF 내부 방향성 합의. 3개 프로젝트의 보완 관계 + ROI 추정 공유 |
| **오너** | 박윤경 |
| **의존성** | BigQuery 접근 |
| **산출물** | |
| | 1. 프로젝트 문서 5종 (00~04) |
| | 2. **상위 30개 상품 기준 월간 예상 추가 GMV** (BigQuery 일매출 + CVR 데이터 기반) |
| | 3. **상위 30개 상품 일평균 PDP뷰 + 현재 CVR 베이스라인** |
| | 4. 기원/현욱 주당 투입 시간 합의 |
| **완료 기준** | TF 리더 리뷰 + 피드백 반영 + ROI back-of-envelope 산출 완료 |
| **상태** | 진행중 |

### ROI 추정 방법 (Back-of-Envelope)

```
예상 추가 GMV = Σ (상위 30개 상품의 일평균 PDP뷰 × 현재 CVR × 예상 CVR 상승률 × 평균 객단가 × 30일)

예상 CVR 상승률: 보수적으로 +1~2pp (Phase 3 A/B 결과로 정밀화)
데이터 소스: BigQuery (일매출, PDP뷰, CVR 이미 존재)
```

### 트래픽 & 표본 크기 사전 산정

> Phase 1에서 확보한 데이터로 아래를 채운다.

| 항목 | 산식/방법 | Phase 0~1에서 확인 |
|------|----------|------------------|
| 상위 30개 상품 일평균 PDP뷰 합계 | BigQuery | _____ 뷰/일 |
| 현재 평균 CVR (베이스라인) | BigQuery | _____% |
| 검출하고 싶은 최소 효과 크기 (MDE) | 보수적 +1pp, 낙관적 +2pp | |
| 필요 표본 크기 (α=0.05, β=0.2) | [Evan Miller calculator](https://www.evanmiller.org/ab-testing/sample-size.html) | _____ 뷰/그룹 |
| 예상 실험 기간 | 필요 표본 ÷ (일평균 뷰 × 50%) | _____ 일 |

**플랜 B**: A/B 인프라 부재 또는 트래픽 부족 시
- Before/After 비교 (시계열 분석) + 베이지안 추론
- 대상 상품군 확대 (30개 → 50개)

## 측정 프레임워크

### 핵심 지표 (North Star)

| 지표 | 정의 | 측정 시점 |

| Phase | 주요 지표 | 목표 |

### 보고 체계

```
주간: Health Score 변화 + Grade 분포 (자동 대시보드)
```
