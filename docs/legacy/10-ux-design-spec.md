## 2. 사이드바 상세 설계

### Section 1: URL 입력

```
┌────────────────────────────────┬──────────┐
│  https://myrealtrip.com/off... │   생성   │
└────────────────────────────────┴──────────┘
  예: https://www.myrealtrip.com/offers/128492
```

- Input: h44px, radius-8px, border 1.5px gray-300
- Placeholder: "마이리얼트립 상품 URL을 붙여넣으세요"
- 버튼: mrt-blue 배경, white 텍스트, font-weight 700
- Focus: border mrt-blue, box-shadow 0 0 0 3px rgba(43,150,237,0.15)
- Error: border mrt-red, helper text 빨간

### Section 2: 카테고리 선택

- UI: Chip Group (4x2 grid), single select
- Chip: h40px, radius-8px, font-size 13px, weight 600
- 기본: white bg, gray-200 border
- Hover: gray-50 bg, gray-400 border
- 선택: mrt-blue-light bg, mrt-blue border+text

```
┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐
│  투어  │ │  티켓  │ │시티패스│ │액티비티│
└────────┘ └────────┘ └────────┘ └────────┘
┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐
│세미패키│ │  교통  │ │  체험  │ │ 서비스 │
└────────┘ └────────┘ └────────┘ └────────┘
```

### Section 3: 샘플 데이터

- UI: Radio List Card (카드형 리스트)
- 각 카드: h~64px, padding 12px 16px, radius-8px, border 1.5px gray-200
- 내부: 이모지 + 상품명(14px/600) + 카테고리+평점(13px/400, gray-500)
- 선택됨: mrt-blue border, mrt-blue-bg 배경, 왼쪽 3px accent
- 클릭 시: 즉시 PDP 렌더링 + 카테고리 Chip 자동 동기화

```
┌─ ● ──────────────────────────────────────┐  ← 선택됨
│  🎫  유니버설 스튜디오 재팬 1일 패스       │
│      티켓 · ★ 4.9 (2,847)                │
└──────────────────────────────────────────┘
```

### Section 4: 블록 구성 정보 (읽기 전용)

- 현재 PDP에 사용된 블록 태그 목록
- 총 블록 수, 예상 비용

## 3. 프리뷰 영역

### 3.1 모바일 뷰 (기본)

폰 프레임 안에 렌더링. MRT 모바일 트래픽 70~80% → 모바일 프리뷰가 실제 경험에 가까움.

- 외부 프레임: w399px, radius-20px, gray-900 배경
- 브라우저 크롬: h36px, traffic light dots + URL bar
- 내부: w375px, overflow-y auto, 커스텀 스크롤바(w4px)
- 그림자: shadow-lg (0 8px 30px rgba(0,0,0,0.12))

### 3.2 데스크톱 뷰

브라우저 윈도우 프레임 안에 렌더링.
- 외부: max-width 768px, radius-12px
- 전환 애니메이션: max-width transition 0.4s cubic-bezier

### 3.3 블록 라벨 오버레이

토글 ON 시 각 `.pdp-block` 상단에:
- 라벨: font-size 10px, weight 700, mono, mrt-blue bg, white text
- 블록 아웃라인: 1px dashed rgba(43,150,237,0.3)

## 4. 상태별 UI

### 4.1 초기 (Empty State)

```
프리뷰 중앙:
        📄
  상품 URL을 입력하거나
  샘플 데이터를 선택하세요
  [ 샘플로 미리보기 → ]
```

### 4.2 로딩 (URL 추출 + AI 생성)

사이드바 프로그레스:
```
⟳  데이터 추출 중...           Step 1/3
████████░░░░░░░░░░░░
```

프리뷰: 스켈레톤 UI (shimmer 애니메이션)
블록 생성되는 대로 progressive rendering

**3단계**: 데이터 추출 → 카테고리 분류 → AI 카피 생성

### 4.3 성공

- 완성된 PDP 렌더링
- 페이드인 애니메이션 (opacity 0→1, translateY 8→0, 300ms)
- 메타 정보: "9개 블록 | TOUR | ~$0.07"

### 4.4 에러

| 에러 | 메시지 |
|------|--------|
| URL 형식 | "유효한 마이리얼트립 상품 URL을 입력해주세요." |
| 404 | "상품 페이지가 존재하지 않거나 접근할 수 없는 URL입니다." |
| 추출 실패 | "상품 데이터를 추출할 수 없습니다." |
| AI 실패 | "AI 카피 생성에 실패했습니다." |

### 4.5 오프라인

- 서버 health check 실패 시 자동 전환
- URL 입력 비활성화, 배너: "⚡ 오프라인 모드 — 샘플 데이터로 동작"
- 카테고리/샘플/뷰모드/라벨토글 정상 동작

## 5. 인터랙션 플로우

### URL 입력 플로우

```
URL 붙여넣기  →  유효성 즉시 검증 (정규식)          즉시
"생성" 클릭   →  프로그레스 바 + 스켈레톤 UI        즉시
Step 1        →  "데이터 추출 중..." (2~3초)        0s
Step 2        →  "카테고리 분류 중..." (1~2초)       ~3s
Step 3        →  "AI 카피 생성 중..." (5~10초)       ~5s
완료          →  전체 PDP 렌더링                    ~15s
```

### 샘플 선택 플로우

- 클릭 즉시 전환 (로컬 데이터)
- 페이드 전환 애니메이션 (150ms out → render → 150ms in)
- 카테고리 Chip 자동 동기화

### 카테고리 변경 시

- 샘플 사용 중: 같은 카테고리 샘플로 자동 전환 or 블록 조합만 변경
- URL 데이터 사용 중: 데이터 유지, 레시피만 변경 → 즉시 리렌더
