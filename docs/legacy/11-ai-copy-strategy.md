> PDP 자동 생성 도구에서 Claude API가 각 블록에 들어갈 카피를 생성하기 위한 전체 전략
> 작성일: 2026-03-16 | 버전: 1.1

## 0. 파이프라인 범위와 제약

### 핵심 목적

**상품소개가 없는 상품에 AI로 이미지+카피를 생성**하여 PDP를 완성한다. 기존 콘텐츠를 재구조화하는 것이 아니라, 부족한 콘텐츠를 새로 만드는 것이 1차 목적.

### 입력 데이터 시나리오

| 시나리오 | 설명 | 생성 품질 | 처리 방법 |
|----------|------|:---:|----------|
| **A. 풍부한 기존 설명** | description 1000자+, 이미지 5장+ | 높음 | 기존 콘텐츠를 블록 구조로 재구조화 + AI 보강 |
| **B. 최소 설명** | 제목, 태그, 포함사항 정도만 | 중간 | AI가 카테고리 컨텍스트 + 포함사항 기반으로 카피 생성 |
| **C. 거의 빈 상품** | 제목 + 카테고리만 | 낮음 | 카테고리 템플릿 기반 최소 카피 + 수동 보완 필요 플래그 |

### 최소 입력 요구사항

```
필수: 상품 제목 + 카테고리
권장: 제목 + 카테고리 + 포함사항 3개 이상 + 소요시간
최적: 위 + 기존 설명 텍스트 + 이미지 1장 이상
```

### AI 생성 금지 필드 (원본 직접 사용)

| 필드 | 이유 |
|------|------|
| 가격 | 실시간 변동, 오류 시 클레임 위험 |
| 취소정책 | 법적 구속력, AI 오류 허용 불가 |
| 좌표/주소 | 정확도 필수 |
| 예약 가능 날짜 | 실시간 데이터 |

## 1. AI 카피 생성 전체 전략

### 1.1 접근법: 2단계 하이브리드 (추천)

**단일 프롬프트 vs 블록별 개별 프롬프트** 비교 후, **2단계 하이브리드**를 추천한다.

| 접근법 | 장점 | 단점 |
|--------|------|------|
| 단일 프롬프트 (12블록 한번에) | API 호출 1회, 비용 최소 | 토큰 한도 초과 위험, 한 블록 실패 시 전체 재생성 필요, 품질 제어 어려움 |
| 블록별 개별 프롬프트 (12회 호출) | 블록별 품질 제어 가능, 실패 시 해당 블록만 재생성 | API 호출 12회, 비용 12배, 총 지연시간 길어짐 |
| **2단계 하이브리드 (추천)** | 컨텍스트 공유 + 블록별 품질 제어 | 구현 복잡도 약간 증가 |

#### 2단계 하이브리드 구조

```
[1단계] 분석 & 전략 수립 (Sonnet 1회)
  입력: 상품 raw 데이터 전체 + 카테고리
  출력: 상품 분석 결과 (핵심 USP, 타겟 고객, 톤앤매너, 키 메시지)
       → 이것을 "카피 브리프"라고 부른다

[2단계] 블록별 카피 생성 (Haiku 병렬 호출)
  입력: 카피 브리프 + 해당 블록의 raw 데이터 + 블록 프롬프트
  출력: 해당 블록의 JSON 데이터
  → 블록별 독립 호출이므로 병렬 처리 가능 (Promise.all)
```

#### 이유

1. **컨텍스트 일관성**: 1단계에서 "이 상품의 핵심은 무엇인가"를 먼저 파악하므로, 2단계의 각 블록 카피가 서로 모순되지 않는다.
2. **비용 효율**: 1단계만 Sonnet, 2단계는 Haiku로 분리하면 비용은 블록별 개별 호출 대비 약 60~70% 절감.
3. **병렬 처리**: 2단계의 블록별 호출은 서로 독립적이므로 `Promise.all`로 병렬 처리 가능. 총 지연시간 = 가장 느린 블록 1개의 응답시간.
4. **실패 격리**: 한 블록이 실패해도 다른 블록에 영향 없음. 해당 블록만 재시도.

### 1.2 모델 선택 전략

| 모델 | 대상 블록 | 이유 |
|------|----------|------|
| **Claude Sonnet** | 1단계 분석, overview, hero(제목/부제) | 분석적 사고, 설득력 있는 카피, 창의적 표현이 필요한 블록 |
| **Claude Haiku** | highlights, inclusions, usageGuide, trustBadges, optionTable, comparison, recommendFor, itinerary, guideProfile, imageGrid | 정형화된 구조, 데이터 변환 중심, 빠른 처리 우선 |

**비용 추정** (1상품 기준):
- 1단계 Sonnet: ~3,000 input / ~2,000 output 토큰
- 2단계 Haiku x 10블록: 블록당 ~1,500 input / ~800 output 토큰
- 총 비용: 약 $0.02~0.04 / 상품

### 1.3 시스템 아키텍처

```javascript
// 호출 흐름 의사코드
async function generatePDPCopy(rawProductData, category) {
  // 1단계: 카피 브리프 생성 (Sonnet)
  const copyBrief = await generateCopyBrief(rawProductData, category);

  // 2단계: 블록별 카피 생성 (Haiku, 병렬)
  const recipe = getRecipe(category); // category-recipes.js에서 가져옴

  const blockPromises = recipe
    .filter(blockType => needsAICopy(blockType)) // cta, socialProof 등 제외
    .map(blockType => generateBlockCopy(blockType, copyBrief, rawProductData, category));

  const blockResults = await Promise.allSettled(blockPromises);

  // 결과 조립
  const blocks = {};
  blockResults.forEach((result, index) => {
    if (result.status === 'fulfilled') {
      blocks[result.value.blockType] = result.value.data;
    } else {
      console.warn(`블록 생성 실패: ${result.reason}`);
      // 실패한 블록은 빈 상태로 두거나 재시도
    }
  });

  return { category, blocks };
}
```

## 2. 카피 톤 & 스타일 가이드

### 2.1 마이리얼트립 브랜드 보이스

```
핵심 정체성: "여행자의 현명한 친구"

— 여행사 느낌이 아닌, 먼저 다녀온 친구가 알려주는 톤
— 권위적이지 않으면서도 전문적인 정보를 전달
— 과장 없이 솔직하게, 하지만 설렘은 잃지 않게
```

| 보이스 속성 | 설명 | DO | DON'T |
|------------|------|----|----|
| **솔직함** | 과장 없이 팩트 기반 | "약 5시간 소요, 도보 이동 많아요" | "환상적인 5시간의 여정" |
| **구체적** | 숫자, 장소명, 시간을 명시 | "블루모스크 내부의 2만 장 이즈닉 타일" | "아름다운 건축물" |
| **편안함** | 딱딱하지 않은 ~해요체 | "가이드가 맛집도 알려줘요" | "가이드의 맛집 추천이 제공됩니다" |
| **행동지향** | 구체적 대상 + 행동 동사, 경험 중심 | "현지인만 아는 골목길을 걸으며" | "현지의 분위기가 느껴지는" |

### 2.2 카테고리별 톤 차이

| 카테고리 | 핵심 감정 | 톤 | 문체 예시 |
|---------|---------|-----|---------|
| **TOUR** | 안심, 편안함 | 따뜻하고 신뢰감 있는 | "현지 12년차 가이드가 숨은 이야기까지 들려줘요" |
| **TICKET_THEME** | 기대감, 실용 | 밝고 에너지 넘치는 | "입장권 1장으로 40개 어트랙션 무제한" |
| **TICKET_TRANSPORT** | 편리, 합리 | 간결하고 명확한 | "공항에서 시내까지, 탭 한 번이면 끝" |
| **TICKET_CITYPASS** | 합리, 똑똑함 | 계산적이고 설득적인 | "3곳만 방문해도 32,000원 절약" |
| **TICKET_EXPERIENCE** | 설렘, 특별함 | 감각적이고 몰입적인 | "도쿄 야경이 발아래 펼쳐지는 90분" |
| **ACTIVITY** | 도전, 짜릿함 | 역동적이고 용기를 주는 | "초보도 OK, 장비 전부 포함" |
| **SERVICE** | 감성, 특별함 | 감성적이고 프리미엄한 | "제주의 빛으로 담아내는 당신만의 순간" |
| **SEMI_PACKAGE** | 편의, 가성비 | 실용적이고 안심시키는 | "항공+숙박+투어, 이 가격에 다 포함" |

### 2.3 금지 표현 (Negative List)

```
절대 금지:
- 검증 불가 최상급: "최고의", "완벽한", "압도적인", "독보적인", "비교불가"
- 가짜 긴급성: "마감 임박!", "한정 수량!", "지금 안 사면 후회!"
- 번역체: "~을 경험하실 수 있습니다", "~가 제공됩니다", "~를 즐기실 수 있는 기회"
- 과장: "꿈같은", "환상적인", "일생일대의" (실제 데이터로 입증되지 않는 한)
- 허위정보: 상품 데이터에 없는 내용을 AI가 만들어내는 것 (할루시네이션)
- 경쟁사 비하: "다른 곳보다 낫다" 류의 비교
- 감정 조작: "이걸 안 하면 여행이 아닙니다"

주의:
- "~세요"체는 hero/CTA에서만 제한적 사용 (남발 금지)
- 이모지는 recommendFor, highlights 아이콘에만 사용 (본문 텍스트에 이모지 금지)
- "약" 표현: 시간/거리에는 OK ("약 5시간"), 가격에는 금지 (정확한 가격만)
```

### 2.4 좋은 카피 vs 나쁜 카피 예시

#### Hero 제목

```
좋은 예:
  "이스탄불 올드 시티, 하루 만에 깊이 있게" (구체적 + 경험)
  "4,200명이 선택한 바르셀로나 워킹투어" (사회적 증거 + 경험)
  "줄 서지 않고 입장하는 사그라다 파밀리아" (핵심 혜택 + 경험)

나쁜 예:
  "최고의 이스탄불 투어" (검증 불가 최상급)
  "이스탄불에서 잊을 수 없는 하루를 보내세요" (모호, 감정 조작)
  "이스탄불 올드 시티 투어 상품" (상품명 반복, 가치 없음)
```

#### Highlights

```
좋은 예:
  "🏛 블루모스크 내부의 2만 장 이즈닉 타일을 전문 가이드와 감상"
  "🚐 호텔 픽업/샌딩 포함, 이동 걱정 제로"
  "📸 가이드가 알려주는 인생샷 포인트에서 사진 촬영"

나쁜 예:
  "아름다운 이스탄불을 만끽하세요" (모호, ~세요 남발)
  "최고의 가이드와 함께하는 투어" (검증 불가 최상급)
  "다양한 장소를 방문합니다" (구체성 없음, 번역체)
```

#### Overview

```
좋은 예:
  "이스탄불의 심장부, 올드 시티를 하루 만에 깊이 있게 만나는 프라이빗 투어입니다.
  유네스코 세계문화유산인 블루모스크와 아야소피아를 전문 가이드의 해설과 함께 둘러보며,
  비잔틴과 오스만 두 제국의 흔적을 동시에 느낄 수 있습니다."

나쁜 예:
  "이 상품은 이스탄불의 여러 유명 관광지를 방문하는 투어 상품입니다.
  전문 가이드가 동행하며 다양한 이야기를 들려줍니다.
  편안하고 즐거운 시간을 보내실 수 있습니다."
  (구체성 없음, 번역체, 감정 조작)
```

## 3. 블록별 프롬프트 설계

### 3.0 1단계: 카피 브리프 생성 프롬프트 (Sonnet)

이 프롬프트의 출력물이 모든 2단계 블록 프롬프트의 공통 입력이 된다.

```

카피 전략 브리프를 만드는 역할이야.

브랜드 보이스: "여행자의 현명한 친구" — 솔직하고, 구체적이고, 편안하고, 행동지향적.

### 3.1 hero 블록 프롬프트 (Sonnet)

```
[SYSTEM]
너는 마이리얼트립의 수석 카피라이터야. 여행 상품 PDP의 히어로 영역
(제목 + 부제)을 작성하는 역할이야.

## 제목 작성 규칙
- 최대 25자 (모바일 1줄)
- 패턴 A: {숫자}+{경험} → "이스탄불에서 꼭 해야 할 3가지, 하루 만에"
- 패턴 B: {사회적 증거}+{경험} → "4,200명이 선택한 바르셀로나 워킹투어"
- 패턴 C: {핵심 혜택}+{경험} → "줄 서지 않고 입장하는 사그라다 파밀리아"
- 금지: "최고의", "완벽한" 같은 검증 불가 최상급

## 부제 작성 규칙
- 최대 50자
- 핵심 스펙을 숫자로 즉시 전달: "약 5시간 · 한국어 · 최대 8명"
- 제목과 정보가 겹치지 않게

## 배지 작성 규칙
- 상품의 가장 큰 신뢰 요소 1~2개 선별
- type: "bestseller" | "new" | "limited" | "discount" | "korean"
- 실제 데이터에 근거한 것만 사용

[USER]
## 카피 브리프
{copy_brief_json}

- 상품명: {product_title}
- 도시: {city}
- 카테고리: {category}
- 소요시간: {duration}

## 6. 품질 보증 체크리스트

### 6.1 자동 검증 가능 항목 (코드로 구현)

```javascript
// 자동 품질 검증 함수
function validateCopyQuality(blockType, data) {
  const errors = [];
  const warnings = [];

  // === 공통 검증 ===

  // 1. 금칙어 검사
  const FORBIDDEN_WORDS = [
    '최고의', '완벽한', '압도적인', '독보적인', '비교불가',
    '꿈같은', '환상적인', '일생일대의',
    '마감 임박', '한정 수량', '지금 안 사면'
  ];
  const allText = JSON.stringify(data);
  FORBIDDEN_WORDS.forEach(word => {
    if (allText.includes(word)) {
      errors.push(`금칙어 발견: "${word}"`);
    }
  });

  // 2. 번역체 패턴 검사
  const TRANSLATION_PATTERNS = [
    /을 경험하실 수 있습니다/,
    /가 제공됩니다/,
    /를 즐기실 수 있는 기회/,
    /하실 수 있으며/,
    /이 가능합니다/
  ];
  TRANSLATION_PATTERNS.forEach(pattern => {
    if (pattern.test(allText)) {
      warnings.push(`번역체 패턴 감지: ${pattern.source}`);
    }
  });

  // === 블록별 검증 ===

  // hero
  if (blockType === 'hero') {
    if (data.title && data.title.length > 25) {
      errors.push(`hero.title 글자수 초과: ${data.title.length}자 (최대 25자)`);
    }
    if (data.subtitle && data.subtitle.length > 50) {
      errors.push(`hero.subtitle 글자수 초과: ${data.subtitle.length}자 (최대 50자)`);
    }
    if (data.badges && data.badges.length > 2) {
      warnings.push(`hero.badges 개수 초과: ${data.badges.length}개 (권장 2개 이하)`);
    }
  }

  // highlights
  if (blockType === 'highlights') {
    if (!data.items || data.items.length < 3 || data.items.length > 5) {
      errors.push(`highlights 항목 수: ${data.items?.length || 0}개 (3~5개 필요)`);
    }
    data.items?.forEach((item, i) => {
      if (item.text && item.text.length > 40) {
        errors.push(`highlights[${i}] 글자수 초과: ${item.text.length}자 (최대 40자)`);
      }
      // 한글 시작 검사 (이모지가 별도 필드이므로 텍스트는 한글로 시작해야)
      if (item.text && !/^[가-힣]/.test(item.text)) {
        warnings.push(`highlights[${i}] 한글로 시작하지 않음`);
      }
    });
  }

  // overview
  if (blockType === 'overview') {
    if (data.paragraphs) {
      if (data.paragraphs[0] && data.paragraphs[0].length > 80) {
        warnings.push(`overview 1단락 글자수: ${data.paragraphs[0].length}자 (권장 80자 이내)`);
      }
      if (data.paragraphs[0] && data.paragraphs[0].startsWith('이 상품은')) {
        errors.push('overview 1단락이 "이 상품은"으로 시작');
      }
    }
  }

  // recommendFor
  if (blockType === 'recommendFor') {
    if (!data.targets || data.targets.length < 3 || data.targets.length > 5) {
      errors.push(`recommendFor 항목 수: ${data.targets?.length || 0}개 (3~5개 필요)`);
    }
    data.targets?.forEach((target, i) => {
      if (target.text && target.text.length > 40) {
        errors.push(`recommendFor[${i}] 글자수 초과: ${target.text.length}자 (최대 40자)`);
      }
    });
  }

  // trustBadges
  if (blockType === 'trustBadges') {
    if (data.badges && data.badges.length > 4) {
      errors.push(`trustBadges 개수 초과: ${data.badges.length}개 (최대 4개)`);
    }
  }

  return { errors, warnings, isValid: errors.length === 0 };
}
```
