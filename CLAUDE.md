# PDP Conversion Design TF — Project Instructions

## 프로젝트 개요

상품소개가 없는 T&A 상품에 AI로 이미지+카피를 생성하여 PDP를 완성하고, 전환율을 높이는 프로젝트.
"상품소개가 없으니까 만든다" + "만들 때 전환을 설계한다"가 핵심 프레이밍.

- **오너**: 박윤경 (T&A Experience Team)
- **TF 구성**: 박윤경(전환 설계) + 기원(AI 이미지+카피 생성) + 현욱(빌더 도구)
- **도메인**: 여행 T&A OTA, 659개 상품, 8개 카테고리 (TOUR, TICKET_THEME/TRANSPORT/CITYPASS/EXPERIENCE, ACTIVITY, SERVICE, SEMI_PACKAGE)
- **핵심 지표**: CVR, ATF 정보 완성도, PDP Health Score

## 동적 전문가 에이전트 시스템

### 핵심 원칙

에이전트는 **고정된 인원이 아니다**. 작업의 성격을 분석해서 가장 적합한 전문가를 **동적으로 구성**한다.

### 에이전트 구성 프로토콜

**Step 1: 작업 분석** — 대상 산출물의 성격을 파악한다.

| 산출물 성격 | 필요한 관점 예시 |
|------------|----------------|
| 기술 구현/아키텍처 | 아키텍트, 시니어 개발자, 보안 엔지니어 |
| UX/UI 설계 | UX 리서처, UI 디자이너, 접근성 전문가 |
| 비즈니스 전략/설득 | PM, 비즈니스 애널리스트, 이해관계자 대변인 |
| 데이터/실험 설계 | 데이터 사이언티스트, CRO 전문가, 통계학자 |
| 카피/콘텐츠 | 카피라이터, 브랜드 전략가, 현지화 전문가 |
| 종합 문서/발표 | 의사결정자 시뮬레이션, 비평가, Devil's Advocate |

**Step 2: 에이전트 설계** — 각 에이전트별로 다음을 정의한다.
- **페르소나**: 직무, 경력, 전문 분야
- **리뷰 렌즈**: 이 에이전트가 집중할 평가 기준 3~5개
- **프로젝트 컨텍스트**: 읽어야 할 docs/ 문서 목록
- **출력 형식**: CRITICAL / HIGH / MEDIUM / LOW + 구체적 액션

**Step 3: 병렬 실행** — Task 도구로 에이전트를 동시에 스폰한다.
- 에이전트 수: 보통 3~5명 (작업 복잡도에 따라 조절)
- 각 에이전트는 독립적으로 리뷰
- 필요 시 에이전트 간 교차 반론 라운드 추가

**Step 4: 종합** — 전체 피드백을 심각도별로 종합한다.

```
종합 보고서 구조:
├── 참여 에이전트 (표: 에이전트명 / 관점 / 핵심 키워드)
├── CRITICAL — 즉시 수정 (프로젝트 성패에 직접 영향)
├── HIGH — 다음 마일스톤 전 해결
├── MEDIUM — 개선 권장
├── LOW — 참고
└── 전문가 공통 평가 (합의 사항 + 핵심 리스크 요약)
```

### 에이전트 생성 규칙

1. **다양성 확보**: 같은 관점의 에이전트를 중복 생성하지 않음. 의도적으로 상충하는 관점 포함
2. **컨텍스트 로딩**: 각 에이전트에게 관련 docs/ 문서를 읽게 함. 전체 문서를 다 읽히지 않고 해당 관점에 필요한 것만
3. **도메인 고정**: 모든 에이전트는 "여행 T&A OTA, 마이리얼트립, TF 3명 규모"라는 맥락을 공유
4. **근거 필수**: 모든 피드백에 구체적 근거(문서 참조, 데이터, 업계 사례) + 액션 아이템
5. **Devil's Advocate 1인 이상**: 중요 의사결정 시 반드시 반대 논거를 전담하는 에이전트 포함

### 자동 트리거 조건

다음 상황에서 에이전트 토론을 **자동 제안**한다 (사용자 승인 후 실행):
1. 문서 신규 작성 또는 대규모 수정 완료 시
2. 설계 의사결정 분기점 (A안 vs B안)
3. Phase 전환 시점의 산출물 검수
4. A/B 테스트 설계/결과 해석
5. 외부 발표/공유 자료 작성 시

### 슬래시 커맨드

- `/review [대상]` — 대상에 맞는 전문가 패널을 동적 구성하여 리뷰
- `/debate [주제]` — 찬성/반대 에이전트를 구성하여 구조화 토론
- `/critique [대상]` — 단일 전문가를 동적 생성하여 심층 비평
- `/synthesize` — 전체 문서 간 일관성 크로스체크
