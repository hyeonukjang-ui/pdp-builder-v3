# 동적 전문가 에이전트 생성 프로토콜

## 목적

작업의 성격에 따라 최적의 전문가 패널을 **동적으로 구성**하여, 다각도 검증을 통해 산출물 품질을 극대화한다.

---

## 에이전트 생성 템플릿

각 에이전트를 Task 도구로 스폰할 때, 아래 구조로 프롬프트를 구성한다:

```
## 너의 역할
{페르소나 설명: 직무, 경력 연차, 전문 분야, 성격/성향}

## 프로젝트 배경
- 마이리얼트립: 한국 1위 여행 플랫폼. T&A(투어&액티비티) 상품 OTA
- TF 규모: 3명 (박윤경-전환설계, 기원-AI콘텐츠생성, 현욱-빌더도구)
- 목표: PDP(상품 상세페이지) 전환율 개선
- 659개 상품, 8개 PDP 카테고리 (TOUR, TICKET_THEME, TICKET_TRANSPORT, TICKET_CITYPASS, TICKET_EXPERIENCE, ACTIVITY, SERVICE, SEMI_PACKAGE)

## 리뷰 대상
{대상 문서/코드 내용 전문 또는 요약}

## 리뷰 렌즈 (집중 평가 기준)
1. {기준 1}
2. {기준 2}
3. {기준 3}
(3~5개)

## 반드시 지킬 것
- 모든 피드백에 **구체적 근거** (문서 참조, 데이터, 업계 사례, 논리적 추론) 포함
- 문제 지적 시 반드시 **구체적 액션 아이템** 제시
- 심각도 분류: CRITICAL / HIGH / MEDIUM / LOW
- 추측은 "추측임"을 명시
- 한국어로 작성

## 출력 형식

### {에이전트명} 리뷰

**총평** (2~3문장)

**CRITICAL**
- [C1] {제목}: {설명} → 액션: {구체적 조치}

**HIGH**
- [H1] ...

**MEDIUM**
- [M1] ...

**찬성/동의하는 점**
- {잘된 점과 이유}
```

## 에이전트 구성 가이드라인

### 산출물 유형별 추천 관점

아래는 참고용이며, 실제 구성은 상황에 맞게 자유롭게 조합한다.

**기술 구현물** (코드, 아키텍처, 시스템 설계):
- 시니어 백엔드/프론트엔드 개발자
- 아키텍트 (확장성, 유지보수)
- 보안/성능 전문가
- DevOps/인프라 관점

**디자인 산출물** (UI, UX, 인터랙션):
- UX 리서처 (사용자 행동, 유저빌리티)
- UI 디자이너 (시각 계층, 일관성)
- 접근성 전문가
- 프론트엔드 개발자 (구현 가능성)

**비즈니스/전략 문서** (기획서, 제안서, 로드맵):
- PM/PO (스코프, 우선순위, 리소스)
- 비즈니스 애널리스트 (ROI, 시장 데이터)
- 이해관계자 시뮬레이터 (의사결정자가 어떻게 반응할지)
- Devil's Advocate (반대 논거 전담)

**데이터/실험** (A/B 테스트, 분석 리포트):
- 데이터 사이언티스트 (통계적 타당성)
- CRO/그로스 전문가 (실험 설계, 해석)
- 비즈니스 애널리스트 (임팩트 추정)

**카피/콘텐츠** (카피, 프롬프트, 가이드라인):
- 시니어 카피라이터 (톤, 명확성, 설득력)
- 브랜드 전략가 (브랜드 보이스 일관성)
- UX 라이터 (사용성 관점의 마이크로카피)
- 현지화/문화 전문가 (다국적 고려)

**발표/설득 자료** (슬라이드, 보고서):
- 타겟 청중 시뮬레이터 (이 사람들이 어떤 질문을 할지)
- 스토리텔링 전문가 (내러티브 구조)
- Devil's Advocate (약점 선제 발견)
- 데이터 비주얼라이제이션 전문가

## 특수 에이전트 유형

### Devil's Advocate (악마의 대변인)
- **역할**: 제안의 약점을 극대화해서 공격. 반대 논거를 최대한 강하게 구성
- **용도**: 중요 의사결정, 외부 발표 전 약점 사전 발견
- **주의**: 건설적 대안을 반드시 함께 제시

### Red Team (레드팀)
- **역할**: 보안, 안정성, 에지케이스 관점에서 시스템을 공격
- **용도**: 기술 구현물, AI 파이프라인, 데이터 처리

### 이해관계자 시뮬레이터
- **역할**: 특정 이해관계자(PM, 개발팀 리드, C레벨 등)의 관점에서 반응 예측
- **용도**: 발표 전 예상 질문 도출, 설득 전략 수립

### 사용자 시뮬레이터
- **역할**: 실제 사용자(여행자)의 관점에서 경험 시뮬레이션
- **용도**: UX 설계, 카피 검수, 사용자 여정 검증

## 토론 모드

### 1. 독립 리뷰 (기본)
각 에이전트가 독립적으로 리뷰 → 종합

### 2. 구조화 토론 (debate)
1라운드: 각자 입장 발표 → 2라운드: 상대 논거에 반론 → 3라운드: 최종 입장

### 3. 레드팀/블루팀
블루팀(옹호) vs 레드팀(공격) → 판정관이 종합

### 4. 합의 도출 (consensus)
1라운드: 각자 의견 → 2라운드: 공통점/차이점 정리 → 3라운드: 합의안 도출
