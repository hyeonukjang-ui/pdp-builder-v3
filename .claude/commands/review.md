# 전문가 패널 리뷰

대상: $ARGUMENTS

## 실행 절차

1. **대상 파악**: 위에 지정된 대상 문서/코드/설계를 읽는다. 대상이 명시되지 않았으면 가장 최근에 작업한 산출물을 대상으로 한다.

2. **에이전트 동적 구성**: 대상의 성격을 분석하여 최적의 전문가 패널을 구성한다.
   - `.claude/agents/protocol.md`의 가이드라인을 참조
   - 에이전트 수: 3~5명 (대상 복잡도에 따라 조절)
   - **반드시 다양한 관점**을 포함 (기술+비즈니스+사용자 등 최소 2개 축)
   - 중요 의사결정이 포함된 경우 **Devil's Advocate 1인 필수**

3. **병렬 실행**: Task 도구로 각 에이전트를 병렬 스폰한다. 각 에이전트에게:
   - 페르소나(직무, 연차, 전문분야, 성향)를 부여
   - 대상 문서 전문 또는 핵심 내용을 전달
   - 리뷰 렌즈(평가 기준 3~5개)를 지정
   - 관련 프로젝트 문서(docs/)를 함께 읽게 함
   - 한국어로 출력하게 함

4. **종합 보고서 작성**: 모든 에이전트 피드백을 수집하여 다음 구조로 종합:

```
## 참여 에이전트
(표: 에이전트명 | 관점 | 핵심 키워드)

## CRITICAL (즉시 수정 — 프로젝트 성패에 직접 영향)
(번호. 제목 + 지적자 + 설명 + 액션)

## HIGH (다음 마일스톤 전 해결)
(번호. 제목 + 지적자 + 설명 + 액션)

## MEDIUM (개선 권장)
(표로 정리)

## 전문가 공통 평가
- 합의 사항 (잘된 점)
- 핵심 리스크 Top 3
- 가장 빠른 개선 포인트
```
