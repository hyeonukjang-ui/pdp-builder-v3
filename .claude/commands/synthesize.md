# 문서 일관성 크로스체크

대상 범위: $ARGUMENTS

## 실행 절차

1. **범위 결정**: 대상이 지정되면 해당 문서들만, 미지정이면 docs/ 전체를 크로스체크한다.

2. **에이전트 구성**: 검증 관점별로 에이전트를 동적 구성한다.
   - **수치/데이터 검증자**: 문서 간 숫자, 비용, 비율 등이 일치하는지
   - **용어/명칭 검증자**: 같은 개념에 다른 용어를 쓰고 있는지
   - **구조/스키마 검증자**: 블록, 필드, API 등의 정의가 일관적인지
   - 필요 시 추가 관점 생성

3. **병렬 실행**: 각 에이전트가 전체 문서를 읽고 불일치를 찾는다.

4. **불일치 보고서 작성**:

```
## 크로스체크 범위
{검사한 문서 목록}

## 불일치 발견 목록

| # | 항목 | 문서 A (내용) | 문서 B (내용) | 정본 | 수정 필요 문서 | 심각도 |
|---|------|-------------|-------------|------|-------------|--------|

## SSoT(Single Source of Truth) 위반
- {정본 규칙에 따른 수정 방향}

## 권장 액션
1. {가장 시급한 수정}
2. ...
```

## 참고
CLAUDE.md의 "Single Source of Truth 규칙" 섹션을 기준으로 어느 문서가 정본인지 판단한다:
- AI 파이프라인 → 04 문서
- 블록 스키마 → 01 문서
- 디자인 토큰 → 09 문서
- 카피 가이드 → 03 문서
